# Epic: wave-1a-consent-schema
**Model:** haiku
**Wave:** 1
**Group:** A (sequential first)

## Goal
Add Prisma models for the ConsentVault feature: ConsentRecord, Recording, RecordingAccess, RecordingRevocation. Add User relations. Run migration.

## Acceptance Criteria
1. `ConsentRecord` model: id, initiatorId (User), participantId (User), status (PENDING/CONFIRMED/REVOKED), gpsLat, gpsLng, bluetoothProof (String?), timestamp (DateTime), expiresAt (DateTime), createdAt
2. `Recording` model: id, consentRecordId (ConsentRecord 1-1), s3Key, drmContentId, drmUrl, duration (Int?), sizeBytes (BigInt?), status (PROCESSING/READY/DELETED), createdAt, deletedAt (DateTime?)
3. `RecordingAccess` model: id, recordingId, userId, isActive (Boolean default true), revokedAt (DateTime?), createdAt
4. `RecordingRevocation` model: id, recordingId, revokedByUserId, reason (String?), createdAt
5. `ConsentStatus` enum: PENDING, CONFIRMED, REVOKED
6. `RecordingStatus` enum: PROCESSING, READY, DELETED
7. User model gets: `consentRecordsInitiated`, `consentRecordsParticipated`, `recordingAccesses` relations
8. `prisma migrate dev --name consent-vault` runs without errors
9. All models use snake_case `@@map` and `@map` decorators matching existing schema style
10. All foreign keys have `onDelete: Cascade` where appropriate

## File Paths
- `services/api/prisma/schema.prisma` — add models and enums
- `services/api/prisma/migrations/` — auto-generated by prisma migrate

## Codebase Context
- Schema file: `services/api/prisma/schema.prisma`
- All models use `@id @default(uuid()) @db.Uuid`, `@map("snake_case")`, `@@map("table_name")`
- DateTime fields use `@default(now()) @map("created_at")`
- All relations have explicit `fields` and `references`
- Run from: `services/api/` directory: `npx prisma migrate dev --name consent-vault`
