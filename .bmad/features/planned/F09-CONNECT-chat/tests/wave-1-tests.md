# Test Specification: F09 Wave 1 — Phoenix Channels Setup

## API Tests
- [ ] WebSocket connection to Phoenix endpoint succeeds with valid JWT
- [ ] WebSocket connection rejected with invalid/expired JWT
- [ ] Channel join "conversation:{id}" succeeds for conversation participants
- [ ] Channel join denied for non-participants
- [ ] Message broadcast received by all participants in < 500ms P95

## Database Tests
- [ ] Conversation created automatically on mutual match
- [ ] ConversationParticipant records created for both users
- [ ] Message persisted with: senderId, conversationId, content, type, createdAt
- [ ] Messages queryable with cursor-based pagination (oldest first, 50 per page)

## Integration Tests
- [ ] Fastify API creates conversation record -> Phoenix can serve the channel
- [ ] JWT generated by Fastify is accepted by Phoenix for channel auth
- [ ] Message sent via Phoenix Channel persisted to PostgreSQL by Fastify consumer
